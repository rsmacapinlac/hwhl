# AGENTS.md

This repo manages a self-hosted homelab with:

- **Terraform** for Proxmox LXC infrastructure.
- **Ansible** for host setup, Docker installation, service deployment, and maintenance.
- **Docker Compose** services running inside LXCs.

Treat `homelab_config/ansible/group_vars/all.yml`, environment files, Terraform variables, and generated tokens/state as sensitive.

## Repository layout

### Scripts

- `bin/ansible-init.sh` — install Ansible and Ansible Galaxy dependencies on the control machine.
- `bin/terraform-init.sh` — install Terraform on the control machine.
- `bin/generate-ssh-keys.sh` — generate homelab SSH keys for Terraform/root and Ansible users.
- `bin/ansible-playbook.sh` — wrapper that runs `ansible-playbook` from `ansible/`.
- `bin/terraform.sh` — wrapper that runs Terraform from `terraform/` with `../homelab_config/terraform.tfvars`.

### Terraform

- `terraform/main.tf` — instantiates all service LXCs from `var.services_containers`.
- `terraform/variables.tf` — Proxmox, global, SSH, template, and container variables.
- `terraform/locals.tf` — computes SSH key data and Ansible inventory group ordering.
- `terraform/ansible-inventory.tf` — generates `homelab_config/ansible/inventory/generated/terraform-hosts.ini`; do not edit that generated file manually.
- `terraform/modules/lxc-container/` — reusable Proxmox LXC module, including nesting, mount points, and `/dev/net/tun` passthrough.

### Ansible

- `ansible/ansible.cfg` — Ansible config; run Ansible from `ansible/` or use the wrapper so inventory and roles resolve correctly.
- `ansible/proxmox-setup.yml` — initial Proxmox setup: repos, templates, optional NFS mounts.
- `ansible/terraform-setup.yml` — Proxmox Terraform user/API token and Terraform tfvars setup.
- `ansible/host-setup.yml` — initial LXC setup as root; creates/configures the `ansible` user.
- `ansible/site.yml` — main service deployment playbook.
- `ansible/maintenance.yml` — service update playbook.
- `ansible/inventory/groups.yml` — group hierarchy, including `managed_hosts`.
- `ansible/roles/` — service and shared roles.

### Homelab config

- `homelab_config/terraform.tfvars` — Terraform environment values.
- `homelab_config/ansible/inventory/generated/terraform-hosts.ini` — generated by Terraform.
- `homelab_config/ansible/inventory/proxmox-lxc-config.yml` — manual inventory additions, vars, and non-Terraform hosts.
- `homelab_config/ansible/group_vars/all.yml` — global Ansible vars and secrets.
- `homelab_config/hosts/` — host-specific service config such as Traefik and Prometheus files.

## Common workflow

Initial setup:

```bash
./bin/ansible-init.sh
./bin/terraform-init.sh
./bin/generate-ssh-keys.sh
```

Infrastructure setup:

```bash
cd ansible
ansible-playbook proxmox-setup.yml
ansible-playbook terraform-setup.yml
```

Terraform operations:

```bash
./bin/terraform.sh plan
./bin/terraform.sh apply
```

After new LXCs are created:

```bash
./bin/ansible-playbook.sh host-setup.yml --limit <host-or-group>
./bin/ansible-playbook.sh site.yml --limit <host-or-group>
```

Deploy or maintain services:

```bash
./bin/ansible-playbook.sh site.yml
./bin/ansible-playbook.sh maintenance.yml
./bin/ansible-playbook.sh site.yml --syntax-check
./bin/ansible-playbook.sh site.yml --check
```
